## Naming Conventions

### Framework output names

The rules generator produces files in `out_rule_generation/version_XX/outputs/`
following this pattern:

| File | Pattern | Example |
|---|---|---|
| Rules file | `<date>_rules_file_from_carrier_list_nr_<version>.tsv` | `2026-01-03_rules_file_from_carrier_list_nr_48B.tsv` |
| Gene list | `list_of_analyzed_genes_science_pipeline_names.json` | (same across versions) |
| Disease-gene metadata | `<date>_disease_gene_metadata.tsv` | `2026-02-16_disease_gene_metadata.tsv` |
| Trace log | `trace_write_file.txt` | (not deployed) |

### Deployed names

Each deploy script (`deployment/deploy_to_s3.sh`) renames the rules file
before uploading. The deployed name varies by environment:

- **Production:** `<date>_rules_file.tsv` — simplified, placed under `rules_main/`.
- **Devel:** `<date>_rules_file_v<pipeline_version>_custom.tsv` — includes pipeline version tag.

The JSON gene list is always deployed with its original name.

The disease-gene metadata TSV keeps its original name and is deployed under
`disease_gene_metadata_main/`.
